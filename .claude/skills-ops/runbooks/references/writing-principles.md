# Principios de Escritura para Runbooks

- **Comandos exactos**, no prosa ("gcloud run services describe alizia-api" no "chequear servicio")
- **Criterios numericos** ("Si X>Y -> Z", no "si te parece raro")
- **Escaneable a las 3am** — headers, code blocks, tablas, no parrafos
- **Sin tribal knowledge** — no "preguntale a Juan", si "Juan P. en PagerDuty"
- **Links directos** a dashboards, no "el dashboard principal"
- **Espanol LatAm** — on-call regional
- **Horario escolar importa** — 10am con clases != 3am
